# 🚀 تعليمات التشغيل السريعة

## الخطوات الأساسية

### 1. تثبيت البرامج المطلوبة
- Node.js من: https://nodejs.org/
- Git من: https://git-scm.com/

### 2. تثبيت المكتبات
افتح Command Prompt أو PowerShell في مجلد `backend-api`:

```bash
npm install
```

### 3. إعداد Firebase

#### أ. إنشاء حساب Firebase
1. اذهب إلى: https://console.firebase.google.com/
2. سجل دخول بحساب Google
3. اضغط "Add project" أو "إضافة مشروع"
4. أدخل اسم المشروع (مثل: elite-engineering)

#### ب. تفعيل Firestore
1. من القائمة الجانبية، اختر "Firestore Database"
2. اضغط "Create database"
3. اختر "Start in test mode"
4. اختر الموقع الأقرب لك

#### ج. تحميل مفتاح الخدمة
1. اضغط على ⚙️ (الإعدادات) أعلى اليسار
2. اختر "Project settings"
3. اختر تبويب "Service accounts"
4. اضغط "Generate new private key"
5. احفظ الملف المحمّل في مجلد `backend-api` باسم: `firebase-adminsdk.json`

### 4. إعداد ملف .env

انسخ ملف `.env.example` إلى `.env`:

```bash
copy .env.example .env
```

افتح ملف `.env` بأي محرر نصوص وعدّل:

```env
FIREBASE_PROJECT_ID=اسم-مشروعك
FIREBASE_CLIENT_EMAIL=البريد-من-ملف-firebase-adminsdk
FIREBASE_PRIVATE_KEY="المفتاح-الخاص-من-ملف-firebase-adminsdk"

JWT_SECRET=اي-نص-عشوائي-طويل-للأمان
JWT_EXPIRATION=7d

PORT=3000
NODE_ENV=development

CORS_ORIGIN=http://localhost:3000,http://localhost:8080
```

### 5. تشغيل السيرفر

```bash
npm run start:dev
```

إذا نجح التشغيل، ستظهر رسالة:
```
🚀 Elite Engineering API is running on: http://localhost:3000/api
```

### 6. إضافة بيانات تجريبية

افتح المتصفح واذهب إلى:
```
http://localhost:3000/api/seed
```

أو استخدم Command Prompt:
```bash
curl -X POST http://localhost:3000/api/seed
```

### 7. اختبار API

افتح المتصفح واذهب إلى:
```
http://localhost:3000/api
```

يجب أن ترى رسالة ترحيب.

## حل المشاكل الشائعة

### مشكلة: Port 3000 is already in use

**الحل:** غيّر PORT في ملف `.env` إلى رقم آخر (مثل 3001)

### مشكلة: Firebase Authentication Error

**الحل:** 
- تأكد من صحة بيانات Firebase في `.env`
- تأكد من وجود ملف `firebase-adminsdk.json`

### مشكلة: npm install فشل

**الحل:**
- احذف مجلد `node_modules`
- احذف ملف `package-lock.json`
- شغّل `npm install` مرة أخرى
